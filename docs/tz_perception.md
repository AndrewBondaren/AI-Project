# ТЗ: Ноды восприятия

## 1. Принцип

Одна геометрическая модель — разные срезы данных в зависимости от типа восприятия.  
Нода восприятия читает геометрию и передаёт LLM только то, что доступно через конкретный канал.

```
Геометрия существует → нода восприятия фильтрует данные → LLM описывает факты
```

LLM не дрейфует — нет зазора между "что есть" и "что LLM говорит что есть".

---

## 2. Типы восприятия

### 2.1 Зрение

Фильтр: прямая видимость, освещённость, дистанция.  
LLM получает: что игрок видит в текущей комнате / зоне.

### 2.2 Слух

Фильтр: звуковое распространение через геометрию.  
Шахты лестниц (uShape, straight) — звуковые коридоры: звук проходит там где зрение не проходит.  
LLM получает: "ты слышишь шаги снизу" — даже если лестница не в прямой видимости.

### 2.3 Камера

Фильтр: другая позиция и угол наблюдения.  
LLM получает: геометрию с точки камеры, а не игрока.

### 2.4 Магическое восприятие

Фильтр: инвертированный — стены прозрачны, видны ауры, потоки, магические объекты.  
LLM получает: тот же геометрический слой + магический слой данных поверх.

---

## 3. Геометрия как основа

Точность восприятия зависит от точности геометрии.  
Реальная 3D-геометрия лестниц (shaft, trapdoor, openings) обеспечивает физически корректное восприятие:
- Слух: звук идёт через shaft
- Зрение: shaft перекрывает прямую видимость
- Магия: тот же shaft может быть каналом магического потока
