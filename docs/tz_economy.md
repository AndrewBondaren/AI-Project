## Торговые пути

Торговые пути строятся поверх существующего графа соединений (`ConnectionNode` / `ConnectionEdge`).

**Принцип:**
- Используются уже существующие рёбра графа дорог
- Если прямого пути нет — может прокладываться новое ребро (новая дорога)
- Pathfinding с весовой функцией по двум критериям:
  1. **Короткость маршрута** — предпочтение более коротким путям
  2. **Опасность** — учитывается при выборе маршрута (опасные участки увеличивают вес)

**Связь с графом соединений:** граф не хранит торговую информацию — торговый pathfinding запрашивает рёбра как есть и считает веса самостоятельно.

**Подробнее:** будет описано в полноценном ТЗ по экономике.
