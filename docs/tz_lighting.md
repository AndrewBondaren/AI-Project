# ТЗ: Освещение (черновик)

> **Статус:** черновик. Реестр зафиксирован; детальная механика (радиус, интенсивность, взаимодействие с погодой) — нет ТЗ.

---

## 1. Реестр типов освещения (`worlds.lighting_type_registry`)

N+1 реестр на уровне мира. Определяет доступные типы источников освещения в зависимости от эпохи и лора.

```json
[
  { "system_lighting": "fire",     "display_name": "Огонь",                "examples": "факела, масляные лампы, костры" },
  { "system_lighting": "electric", "display_name": "Электроэнергия",       "examples": "фонарные столбы, прожекторы" },
  { "system_lighting": "magical",  "display_name": "Магическое",           "examples": "магические шары, светящиеся кристаллы, руны" },
  { "system_lighting": "advanced", "display_name": "Продвинутое",          "examples": "голографические панели, биолюминесценция, плазма" }
]
```

Миры с разной эпохой включают только релевантные типы: средневековый мир — `fire` + `magical`; sci-fi — `electric` + `advanced`.

## 2. Освещение дорог

Дороги и тротуары (`ConnectionEdge`) имеют собственное освещение через поле `lighting_type`.

**Качество источников освещения зависит от `economic_tier`** поселения (`CitySkeleton.economic_tier`):

| `economic_tier` | Ожидаемое качество освещения |
|---|---|
| Бедный | редкие факела, масляные лампы низкого качества |
| Стандартный | регулярные фонари, нормальное покрытие |
| Качественный | плотное покрытие, качественные источники |
| Исключительный | полное покрытие, возможны продвинутые типы |

Конкретные параметры (плотность столбов, радиус света, материал) — резолвятся через `building_template_registry` по `structure_type="lamp_post"` + `economic_tier`.  
`lighting_type` при этом определяет физику источника, `economic_tier` — его качество и плотность.

---

## 3. Применение

Освещение как типизированная ссылка используется везде где есть источник света:

| Контекст | Поле |
|---|---|
| Дорога / тротуар (`ConnectionEdge`) | `lighting_type` → ref → `lighting_type_registry` |
| Комната (`NamedLocation`) | будет описано в ТЗ по интерьеру |
| Объект вдоль ребра (фонарный столб как `street_object`) | `lighting_type` в параметрах объекта |
